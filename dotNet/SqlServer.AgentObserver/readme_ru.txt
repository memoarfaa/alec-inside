SqlServer.AgentObserver отслеживает состояние SQL Server Agent и его заданий(Job).
C заданной периодичностью проверяет запущен ли сервис SQL Agent и если нет, то запускает его.
Если сервис запущен, то проверяется состояние всех его заданий. К сожалению, время запуска задания не возможно
получить до тех пора пока задение не будет завершено. Единственное что известно, это время
следующего запуска задания по расписанию. Поэтому логика определения зависших заданий такая:
если разность между текущим временем и планируемым временем следующего запуска больше установленного значения
в минутах, то значит задание зависло. Если обнаружено хотябы одно зависшее задание, то происходит рестарт SQL Agent.

Установка.
1. Для запуска требуется .Net Framework 2.0
2. Программа устанавливается как служба Windows на тот же компьютер, где установлен SQL Server.
3. Скопируйте файлы в нужную папку и запустите install.bat, он зарегистрирует службу в ОС.
4. При установке службы будет показано окно ввода логина и пароля. От имени этого пользователя служба будет работать.
 Для избежания проблем указываем учетную запись администратора компьютера. При этом имя для локальной учетной записи 
должно быть указано в формате ".\Имя", а для доменной как обычно "Домен\Имя" - естественно без ковычек.
5. Если все сделано правильно служба должна появиться в списке под именем Agent Observer for SQL Server.

Настройка конфигфайла.

В конце конфигфайла есть строчки:
	<appSettings>
		<add key="MissingTimeToNextJobRun" value="1" />
		<add key="InquiryPeriod" value="1" />
		<add key="SQLServerInstanceName" value="" />
	</appSettings>
1. MissingTimeToNextJobRun (целое число) - если количество минут на которые задание просрочило свой очередной запуск
больше этого значения, то считается, что задание повисло.
2. InquiryPeriod (целое число) - определяет период в минутах между опросами состояния SQL Agent и его заданий.
Наверное, имеет смысл задвать его не меньше чем 1/3 или 1/4 от MissingTimeToNextJobRun.
3. SQLServerInstanceName - имя экземпляра SQL Server для которого работает агент. Если экземпляр по-умолчанию, то пустая строка.
Пример: для SQL Server SVNSRV\SQL2005 будет так <add key="SQLServerInstanceName" value="SQL2005" />, может и так тоже нормально
<add key="SQLServerInstanceName" value="SVNSRV\SQL2005" /> - я не пробовал.
